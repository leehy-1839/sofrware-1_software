# 과제2_프로젝트 관리 계획서
## 프로젝트 주제 : 공강 스터디 매칭 서비스
### 1. 개요
1-1. 문서 목적
본 문서는 '공강 스터디 매칭 서비스' 개발 프로젝트의 전반적인 계획을 정의하며, 프로젝트 진행 과정에서 발생하는 세부 사항의 지표로 삼는 것을 목적으로 한다.
1-2. 프로젝트 개요
정의: 학교 내 빈 강의실 정보를 실시간으로 확인하고, 공강 시간이 일치하는 학생끼리 스터디를 매칭해주는 플랫폼.
주요 기능: 빈 강의실 조회, 공강 시간 공유, 스터디 매칭 API, 매칭 게시판.

### 2. 개발 계획
개발 절차 모형: 요구사항이 자주 변경될 가능성을 고려하여, 사용자 요구사항을 빠르게 반영할 수 있는 Agile 방법론 중 SCRUM 모델을 채택함.
분석/설계 방법론: 클래스 단위의 모듈화와 확장성이 용이한 객체지향 방법론을 사용.
WBS (작업 분류 체계):요구분석: 계획서 검토, 기존 유사 서비스 조사, 기능 정의.
설계: 시스템 구조 설계, 상세 설계.코딩: 기능 구현, 서버/DB 구축.테스트: 단위/통합 테스트, 매뉴얼 작성.

### 3. 팀 구성
3.1 팀 구조
팀장의 리더십을 기반으로 하되, 모든 멤버가 상호 협력하고 개발 활동을 투명하게 확인하는 수평적 팀 구조를 채택.
### 4. 품질 관리 적용 기법팀 미팅 계획:
Sprint Planning Meeting: 매주 금요일, 목표 설정.
Daily Scrum Meeting: 매일 30분, 일일 이슈 및 계획 공유.
Sprint Review & Retrospective: 스프린트 종료 시점, 성과 리뷰 및 회고.
변경 관리: 기능 변경 시 팀원 전체 합의 및 다수결 원칙 적용, 의견 충돌 시 팀장의 조정 권한 강화.
산출물 관리: 문서명에 [팀명]과 날짜, 버전을 명시하여 버전 관리 수행
